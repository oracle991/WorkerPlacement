# AGENTS.md

## 概要
- プロジェクトはワーカープレイスメント・プロトタイプ `辺境開拓団(仮)`。
- プレイヤーは辺境開拓団のリーダーとして、個性あるワーカーを村周辺のスポットへ派遣する。
- ソロプレイのデジタルボードゲームで、「置く -> 解決 -> 補充/維持」のループが中心。
- 1プレイは約15-20分、12ラウンドで完結する静的ブラウザゲームを目指す。
- 最終目標は、標準12ラウンド以内に村の繁栄度を目標値30まで上げること。
- 敗北条件は、最終ラウンド終了時の繁栄度不足、またはワーカー0人。

## ゲームループ
- 1ラウンドは準備、配置、解決、維持の4フェーズ。
- 準備では季節イベント、各スポットのイベント、ダイス目標値をプレビューする。
- 配置ではワーカーをドラッグ&ドロップ、またはクリック選択でスポットへ置く。
- 配置前に成功率、参照ステータス、報酬、リスクを見せる。
- 解決はスポット順に自動再生し、クリックやSpaceで演出スキップ可能にする。
- 判定式は `2d6 + 対応ステータス + 個性補正 >= スポット難易度`。
- 結果は大成功、成功、失敗、大失敗の4段階。
- 維持では食料消費、疲労回復、負傷回復判定、繁栄度更新、勝敗判定を行う。
- ランダム要素は使うが、「知らされていない失敗」は作らない。

## 主要ルール
- ワーカーのステータスは筋力、器用、知恵、魅力の4種で、値は1-5。
- ワーカーは1-2個の個性を持ち、判定や結果に条件付き補正を与える。
- 状態は経験値、疲労、負傷を扱う。成長要素はM5まで後回し。
- 資源は食料、木材、鉱石、金貨、繁栄度。
- 初期スポットは農場、森林、鉱山、市場、工房、集会所。
- 第2季節で遺跡、第3季節で祠を解禁する。
- 工房/施設建設が繁栄度の主な源。
- 集会所は新ワーカー勧誘を担当する。
- 季節は春、夏、秋、冬。冬は農場産出半減や食料消費増で難化する。

## 技術スタック
- フレームワークは Vite + TypeScript + Phaser 3。
- テストは Vitest を使う。
- RNGは依存追加より、自前のシード付き `mulberry32` 実装を優先する。
- 状態管理ライブラリは入れない。
- ゲームロジックは Phaser 非依存の純 TypeScript として `src/core/` に置く。
- Phaser Scene は描画、入力、演出、UI接続だけを担当する。
- 個性、スポット、イベントテーブル、季節補正は `src/data/` のJSON/TS定義へ分離する。
- セーブは当面 `localStorage` で中断再開のみ。
- ホスティングは完全静的サイト。`vite build` の成果物を GitHub Pages / Cloudflare Pages / Netlify に置ける構成にする。
- 対応環境は最新のデスクトップ Chrome / Edge / Firefox。
- 1280x720基準の固定アスペクト + Phaser `Scale.FIT` を前提にする。

## UI/ビジュアル
- 2D見下ろし気味の斜め俯瞰ボードゲーム風。
- プロトタイプは色分け図形と絵文字/アイコンのプレースホルダで開始してよい。
- 後で温かみのあるファンタジー調ピクセルアート、またはフラットイラストへ差し替える。
- 画面中央は村マップ、下部はワーカー手札トレイ、上部は資源/ラウンド/季節バー。
- ホバーでワーカー/スポット詳細、成功率、報酬テーブルを出す。
- 最重要演出は2個のダイスロール。Phaser Tweenで軽量に実装する。
- 日本語UIはWebフォント読み込みを使う。

## マイルストーン
- M0: Vite + TS + Phaser 3 + Vitest の雛形と `core / scenes / data` 構成。
- M1: 純TSのコアロジック、判定式、資源、ラウンド進行、シード付き乱数。
- M2: 盤面、プレースメントUI、ドラッグ&ドロップ、ツールチップ、成功率表示。
- M3: ダイス演出、4段階結果、資源フロートテキスト、演出スキップ。
- M4: 季節、スポット解禁、勝敗判定、タイトル/リザルトで12ラウンド完走。
- M5: 個性10種以上、イベント拡充、勧誘、負傷/疲労、施設バリエーション、バランス。
- M6: アセット/SE/BGM、難易度、localStorage、静的デプロイ。
